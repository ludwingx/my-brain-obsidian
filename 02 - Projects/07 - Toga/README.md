# Toga (Cafetería)

## 🧠 Descripción
Ecosistema digital propuesto para una cafetería que integra una landing page comercial, menús interactivos por QR y un sistema ERP para gestionar operaciones internas (inventario, ventas, producción).

## 🎯 Objetivo
Crear una experiencia híbrida moderna en cafetería: digitalizar la visita del cliente final (pedidos y pagos vía QR) y centralizar la gestión operativa del negocio.

## ⚙️ Stack Preferido (A definir)

| Tecnología | Propósito |
|---|---|
| Frontend | Next.js / React |
| Backend | API en Next.js o Node/Django |
| Base de Datos | PostgreSQL / MongoDB |

## 🔑 Componentes del Sistema
1. **Landing Page:** Vitrina web, contacto, historia.
2. **Experiencia Cliente (Front):** Menú digital vía QR, carrito virtual, subida de comprobantes.
3. **ERP Interno (Back-office):** Control de inventario de cafetería, módulos de producción, compras y facturación.
